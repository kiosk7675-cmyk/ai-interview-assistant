# 操作系统八股文

## 进程、线程、协程的区别？
| 特性 | 进程 | 线程 | 协程 |
|-----|------|------|------|
| 资源分配 | 独立地址空间 | 共享进程资源 | 共享线程资源 |
| 调度 | OS 调度 | OS 调度 | 用户态调度 |
| 切换开销 | 大（切换页表等） | 中（切换寄存器等） | 小（用户态切换） |
| 通信 | IPC（管道/共享内存/信号量） | 共享内存（需同步） | 共享内存（无需同步） |
| 并发 | 多进程并发 | 多线程并发 | 协作式并发 |

## 进程间通信（IPC）方式？
- **管道（Pipe）**：半双工，父子进程间通信，数据流式
- **命名管道（FIFO）**：有名字的管道，无亲缘关系进程也可通信
- **消息队列**：内核中的消息链表，按类型读取
- **共享内存**：最快的方式，多进程映射同一块内存，需配合同步机制
- **信号量**：计数器，用于进程/线程同步（P/V 操作）
- **信号**：异步通知机制（SIGKILL, SIGTERM, SIGUSR1 等）
- **Socket**：可用于不同主机间通信，最通用

## 进程调度算法？
- **FCFS（先来先服务）**：非抢占式，简单但平均等待时间长
- **SJF（短作业优先）**：非抢占式，平均等待时间最短，但可能饥饿
- **SRTF（最短剩余时间优先）**：SJF 的抢占版本
- **RR（时间片轮转）**：抢占式，每个进程分配时间片，用完排队
- **优先级调度**：抢占/非抢占，可能导致低优先级饥饿
- **多级反馈队列**：多个优先级队列，时间片随优先级降低而增大，兼顾响应和吞吐

## 死锁的四个必要条件？
1. **互斥**：资源一次只能被一个进程使用
2. **持有并等待**：持有资源同时等待其他资源
3. **不可抢占**：资源不能被强制夺走
4. **循环等待**：进程间形成环形等待链
- **预防**：破坏任一条件（如资源有序分配破坏循环等待）
- **避免**：银行家算法（分配前检查是否安全）
- **检测恢复**：检测死锁后终止进程或抢占资源

## 用户态和内核态？
- **用户态**：运行用户程序，受限访问（不能直接操作硬件、访问内核内存）
- **内核态**：运行内核代码，完全权限（访问所有内存、操作硬件）
- **切换方式**：系统调用（主动）、中断（外部）、异常（被动）
- **切换开销**：保存/恢复寄存器、切换栈、可能刷新 TLB 和缓存

## 常见系统调用？
- **文件操作**：open, read, write, close, lseek
- **进程控制**：fork, exec, wait, exit
- **进程通信**：pipe, shmget, mmap, socket
- **内存管理**：brk, mmap, munmap
- **信号**：kill, signal, sigaction

## 虚拟内存原理？
- **定义**：每个进程有独立的虚拟地址空间，通过页表映射到物理内存
- **MMU**：内存管理单元，硬件完成虚拟地址到物理地址的转换
- **TLB**：Translation Lookaside Buffer，页表缓存，加速地址转换
- **缺页中断**：访问的虚拟页不在物理内存中 → 触发缺页中断 → 从磁盘换入 → 更新页表
- **好处**：进程隔离、内存保护、按需分配、支持大于物理内存的程序

## 页面置换算法？
- **OPT（最优）**：淘汰最远将来才使用的页，理论最优无法实现
- **FIFO**：淘汰最早进入的页，简单但可能淘汰常用页（Belady 异常）
- **LRU（最近最少使用）**：淘汰最久未访问的页，性能好但实现开销大
- **Clock（时钟算法）**：LRU 近似，用引用位+循环指针，实现简单
- **Linux 采用**：改进的 Clock 算法（双指针 + 引用位 + dirty 位）

## 进程同步机制？
- **互斥锁（Mutex）**：保证同一时刻只有一个线程访问临界区
- **信号量（Semaphore）**：计数器，控制同时访问资源的线程数
- **条件变量（Condition Variable）**：配合互斥锁，等待/通知机制
- **读写锁**：读共享、写独占，适合读多写少场景
- **自旋锁**：忙等待（CPU 空转），适合短临界区
- **乐观锁**：CAS（Compare And Swap），无锁并发，适合低冲突场景

## 文件系统原理？
- **inode**：存储文件元数据（权限、大小、时间戳、数据块指针），不存文件名
- **目录**：目录项 = 文件名 + inode 号，目录本身也是文件
- **硬链接**：多个目录项指向同一 inode，删除一个不影响其他（不能跨文件系统）
- **软链接（符号链接）**：独立 inode，存目标文件路径（可跨文件系统）
- **磁盘结构**：超级块 → inode 位图 → 数据块位图 → inode 表 → 数据块

## Linux 常用命令？
- **进程**：ps aux, top, htop, kill -9 PID, lsof -i :port
- **内存**：free -h, vmstat, pmap PID
- **磁盘**：df -h, du -sh *, iostat
- **网络**：netstat -tlnp, ss -tlnp, tcpdump, curl, wget
- **文件**：find, grep, tail -f, awk, sed, wc
- **权限**：chmod 755, chown user:group, sudo

## select、poll、epoll 区别？
| 特性 | select | poll | epoll |
|-----|--------|------|-------|
| 最大连接数 | 1024（FD_SETSIZE） | 无限制 | 无限制 |
| 数据结构 | bitmap | 链表 | 红黑树+就绪链表 |
| 内核态→用户态拷贝 | 每次全量拷贝 | 每次全量拷贝 | 只拷贝就绪 fd |
| 时间复杂度 | O(n) | O(n) | O(1)（就绪事件） |
| 触发方式 | 水平触发 | 水平触发 | 水平/边缘触发 |
| 适用场景 | 少量连接 | 中等连接 | 大量连接（C10K） |
