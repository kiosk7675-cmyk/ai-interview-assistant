# 产品经理与项目管理面经

## 产品思维

### 需求分析
需求来源：用户反馈、数据分析、竞品分析、业务目标、技术驱动。需求评审：评估可行性（技术/资源/时间）、优先级排序（KANO 模型：基本型/期望型/兴奋型）。伪需求识别：区分"用户说的"和"用户真正需要的"，追问 5 个 Why。

### 用户研究方法
定性：用户访谈（深度理解动机）、可用性测试（观察操作行为）、焦点小组。定量：问卷调查（大样本统计）、A/B 测试（数据驱动决策）、埋点分析（用户行为路径）。用户画像：基于真实数据构建典型用户特征标签。

### 竞品分析
直接竞品（同赛道）、间接竞品（替代方案）、潜在竞品。分析维度：功能对比、用户体验、商业模式、市场份额、技术架构。输出：竞品矩阵、SWOT 分析、差异化策略。

## 产品设计

### 产品需求文档（PRD）
核心内容：背景与目标、用户场景、功能需求（用例图/流程图）、非功能需求（性能/安全/兼容性）、数据埋点方案、灰度/上线计划。验收标准：可测试、可衡量。

### 原型设计
低保真：线框图（快速验证思路）。高保真：交互原型（Figma/Axure）。设计原则：一致性、可发现性、容错性、效率。信息架构：导航设计（全局/局部/面包屑）、内容组织（卡片分类法）。

### 用户体验（UX）
尼尔森十大可用性原则：系统状态可见、系统与现实匹配、用户控制与自由、一致性与标准、错误预防、识别而非回忆、灵活高效、美学与极简、帮助用户识别和恢复错误、帮助文档。

## 数据驱动

### 核心指标体系
AARRR 漏斗：获取（Acquisition）→ 激活（Activation）→ 留存（Retention）→ 变现（Revenue）→ 传播（Referral）。北极星指标：衡量产品核心价值的单一指标。DAU/MAU、留存率（次日/7日/30日）、转化率、ARPU、LTV。

### A/B 测试
流程：提出假设 → 设计实验 → 确定样本量和周期 → 分流 → 收集数据 → 统计检验（p-value < 0.05）。注意：同时进行多实验需正交分流、新奇效应、辛普森悖论。

## 项目管理

### 敏捷开发
Scrum 框架：Sprint（2-4 周迭代）、Sprint Planning、Daily Standup、Sprint Review、Sprint Retrospective。角色：Product Owner（需求优先级）、Scrum Master（流程保障）、Development Team（自组织）。Backlog 管理：User Story + Acceptance Criteria + Story Point 估算。

### 看板方法
可视化工作流（To Do → In Progress → Code Review → Testing → Done）。WIP 限制：每列设置在制品上限，防止瓶颈堆积。周期时间（Lead Time）：需求提出到上线的总时长。

### 风险管理
识别风险 → 评估影响和概率 → 制定应对策略（规避/转移/减轻/接受）。常见风险：需求变更频繁、技术方案不确定、关键人员离职、外部依赖延期。风险登记册持续跟踪。

## 软技能面试

### STAR 法则
Situation（情境）：什么背景下。Task（任务）：你要做什么。Action（行动）：你具体怎么做。Result（结果）：取得了什么成果（量化）。每个行为面试题都应按此结构回答。

### 常见行为面试题
- "讲一个你解决过的最困难的问题"：展示分析能力和韧性
- "如何处理团队分歧"：展示沟通和协调能力
- "失败的项目经历"：展示复盘和成长
- "如何说服别人接受你的方案"：展示影响力和数据思维
- "时间冲突如何优先级排序"：展示判断力和沟通能力

### 沟通与表达
向上沟通：结论先行、数据支撑、给选项而非只给问题。跨部门协作：理解对方目标、寻找共同利益、明确分工和交付物。技术方案汇报：问题→方案→权衡→计划，非技术人听得懂。
