# 网络安全面经

## Web 安全

### SQL 注入
攻击者在输入中嵌入 SQL 语句，绕过验证或操作数据库。类型：联合查询注入、盲注（布尔/时间）、堆叠查询。防御：参数化查询（预编译）、ORM 框架、输入校验白名单、最小权限账户、WAF。

### XSS（跨站脚本攻击）
反射型：恶意脚本在 URL 中，服务器原样返回。存储型：恶意脚本存入数据库，所有用户访问时触发。DOM 型：前端 JS 直接读取 URL 参数渲染。防御：输出编码（HTML/JS/URL）、CSP（Content-Security-Policy）、HttpOnly Cookie、输入校验。

### CSRF（跨站请求伪造）
攻击者诱导用户在已登录状态下访问恶意页面，自动发送请求执行操作。防御：CSRF Token（服务端生成、表单提交验证）、SameSite Cookie（Strict/Lax）、Referer/Origin 检查、关键操作二次验证。

### SSRF（服务端请求伪造）
攻击者让服务器发起请求访问内网资源。防御：URL 白名单、禁止内网 IP 访问、限制协议（只允许 http/https）、禁用重定向、独立网络隔离。

### 文件上传漏洞
绕过方式：修改扩展名、MIME 类型伪造、00 截断、双写绕过。防御：白名单扩展名、文件内容检测（Magic Bytes）、重命名文件、存储到非 Web 目录、图片二次渲染。

## 认证与授权

### JWT vs Session
Session：服务端存储，Cookie 携带 Session ID，有状态。JWT：无状态，Token 包含用户信息（Header.Payload.Signature），客户端存储。JWT 优势：无服务端存储、跨服务传递。JWT 劣势：无法主动失效、Payload 明文（不可存敏感信息）。JWT 安全：短过期+Refresh Token、签名算法校验、不存敏感信息。

### OAuth 2.0
授权框架，四种模式：授权码模式（最安全，服务端应用）、隐式模式（已不推荐）、密码模式（高度信任）、客户端凭证模式（服务间通信）。PKCE 扩展：防止授权码被截获，适用于移动端/SPA。

### RBAC / ABAC
RBAC（基于角色的访问控制）：用户→角色→权限，简单直观。ABAC（基于属性的访问控制）：用户属性+资源属性+环境属性+操作，灵活但复杂。最小权限原则：默认拒绝，按需授权。

## 密码学

### 对称加密 vs 非对称加密
对称：AES（128/192/256位），加解密用同一密钥，速度快，密钥分发是难点。非对称：RSA/ECDSA，公钥加密私钥解密，速度慢，解决密钥分发。实际使用：非对称协商密钥 → 对称加密数据（TLS 流程）。

### 哈希与签名
哈希：SHA-256/SHA-3，单向、抗碰撞。密码存储：加盐（Salt）+ 慢哈希（bcrypt/scrypt/Argon2）。数字签名：私钥签名 + 公钥验证，确保完整性+不可抵赖。数字证书：CA 签发，绑定公钥与身份。

## 安全运维

### 渗透测试流程
信息收集 → 漏洞扫描 → 漏洞利用 → 权限提升 → 横向移动 → 报告编写。工具：Nmap（端口扫描）、Burp Suite（HTTP 代理）、SQLMap（SQL 注入）、Metasploit（漏洞利用）。

### 日志审计与应急响应
关键日志：访问日志、错误日志、认证日志、操作日志。异常检测：异常登录时间/IP、高频请求、权限变更。应急响应：隔离→止血→取证→分析→修复→复盘。攻击溯源：时间线还原、IOC 提取、威胁情报关联。

### 等保与合规
等级保护（等保 2.0）：五级分类，二级以上需测评。GDPR：欧盟数据保护，用户同意、数据可携、被遗忘权。个人信息保护法（PIPL）：中国数据保护，最小必要原则、知情同意。

### API 安全
认证：API Key / OAuth 2.0 / Mutual TLS。限流：令牌桶/漏桶，防止暴力攻击。输入校验：参数类型、长度、范围。HTTPS：TLS 1.2+，HSTS。敏感数据脱敏：日志中隐藏手机号/身份证/密码。
